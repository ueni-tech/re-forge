# forge-mino-01 — 低凝集の解消: データとロジックの距離を縮める

ミノ駆動本 3章・6章・7章で扱われる「データとロジックが分散した低凝集コード」を、3段階の距離の縮め方で身につける forge。

**書籍系列 forge の1本目**。命名規約は `forge-{書籍タグ}-{連番}-{テーマ}`。

## 練習の進め方

各 heat は2段階構成。詳細はリポジトリ直下の README を参照。

1. **段階A**: `problem.md` だけ読んで `kata.ts` を実装する
2. **段階B**: `spec.md` を開いて自分の判断と突き合わせ、テストを通す

```bash
npx vitest forge-mino-01-low-cohesion
```

## この forge は2系統で構成される

| サブディレクトリ | 演習スタイル | 訓練するスキル |
|----------------|------------|--------------|
| `./heat-N/` | **白紙からクラス実装** (forge 系) | 不変条件と振る舞いの実装 |
| `./refactor/heat-N/` | **既存コードからの抽出** (refactor 系) | 抽出範囲の判断、API 設計、呼び出し側の書き換え |

同じテーマ (Price / Stock / OrderItems) を 2つの角度から経験することで、**「設計とは何か」が立体化**する。

推奨順序: **先に forge 系を解く → refactor 系に進む**。

「ゴール像 (理想形)」を白紙実装で身につけてから、「現実の散らかったコードから理想形へ橋を架ける」refactor 系に進むと、両方の学びが深まる。

ただし csk が異なる順で進めるなら、それでよい。判断理由を言語化することが大事。

## この forge を貫く一本の軸

> **「データを操作する権利を、誰が持つべきか」**

低凝集コードは、データの所有者と操作者が分離している。リファクタリングの本質は、**操作の権利をデータの近くに移すこと**。Value Object、ファーストクラスコレクション、Tell, Don't Ask — どれもこの原則の別表現。

## heat 構成

| heat | テーマ | 距離の縮め方 | ミノ駆動本対応 |
|------|--------|------------|----------------|
| heat-1 | `Price` | 関数 → メソッド: データとロジックを束ねる | 3章・6章（凝集度） |
| heat-2 | `Stock` | バラのプロパティ → 値オブジェクト | 3章（Value Object）・6章 |
| heat-3 | `OrderItems` | 配列 → コレクション: 操作を内側に閉じる | 7章（ファーストクラスコレクション） |

## 実務との対応

3 heat すべて EC ドメインの例で構成している。レガシー jQuery / PHP 環境では以下のような形で頻出するはず:

- 価格・税・割引: グローバルな `formatYen()`, `calcTax()` 関数群
- 在庫管理: `if (stock > stockLimit) ...` のようなプリミティブ条件分岐
- 注文・カート: `cart.items.filter(...).reduce(...)` のような外部からの直接操作

これらを「データに振る舞いを持たせる」設計に書き換えるのがこの forge のテーマ。

## JSDoc【契約】を書く考え方

forge-01-closure の README を参照。本 forge でも同じ4問テーブルに従う。

特にこの forge では **【設計の読解】が重要**。なぜなら低凝集の解消は「型シグネチャに現れない選択」が多いから:

- なぜ getter を生やさないか
- なぜプリミティブを返さず別の値オブジェクトを返すか
- なぜコレクションをそのまま外に出さないか

これらは型では表せず、JSDoc に書くしかない設計判断。

## 観察ポイント: 3 heat で「外から見えなくなったもの」

各 spec.md の最後で対比表を入れている。3 heat を通じて読むと、**カプセル化が階段状に進む様子**が見える。

| heat | 何を束ねた？ | 外から見えなくなったもの |
|------|------------|------------------------|
| heat-1 | データ + 操作 | 計算ロジックの所在 |

| heat-3 | コレクション + 操作 | 配列そのもの |
