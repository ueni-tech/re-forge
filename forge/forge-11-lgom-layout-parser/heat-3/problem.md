# parseLGomLayout

難易度: ★★★

CMS が出力する `lGomLayout` フィールドの生 HTML 文字列（例: `"1行目：社名or店名<br>2行目：役職＆氏名"`）を受け取り、フィルタタグの配列（例: `["会社名・店名", "氏名"]`）を返す関数を書く。

> **背景**: 住所印商品一覧の「印面内容」フィルタでは、各商品カードの `data-filter-inmen-content` 属性にタグ CSV（`"会社名・店名,氏名"` など）を埋め込む。このタグ CSV を PHP が描画時に生成するために作ったのがこのパーサー。入力データは `goods_address_text.json` の `lGomLayout` フィールドで、`<br>` 区切りの行ラベル文字列。MVP では同じロジックを JS で書いていたが、本番化でサーバー側（PHP）に移した。

## 実装する関数

`parseLGomLayout` を自分で宣言・実装する（`export` する名前は固定）。ヘルパー関数が必要なら同じファイルに書く。**heat-1・heat-2 からの import は行わない**（各 heat は独立）。

## 変換仕様

| 入力 | 出力 |
|------|------|
| `"1行目：社名or店名<br>2行目：役職＆氏名"` | `["会社名・店名", "氏名"]` |
| `"1行目：郵便番号＆住所<br>2行目：社名or店名<br>3行目：TEL＆FAX"` | `["住所", "会社名・店名", "TEL", "FAX"]` |
| `"1行目：郵便番号＆住所<br>2行目：社名or店名（大）<br>3行目：TEL<br>4行目：FAX"` | `["住所", "会社名・店名", "TEL", "FAX"]` |
| `"1行目：住所<br>2行目：住所"` | `["住所"]` |
| `""` | `[]` |
| `null` / `undefined` | `[]` |

## 合意済み仕様

1. **正規化**: `<br>` / `<br/>` / `<br />` を改行（`\n`）に置換する。半角 `&` を全角 `＆` に変換する。前後の空白を trim する。
2. **行分割**: `\n` で分割する。
3. **行ラベル抽出**: 各行から `"N行目："` プレフィックスと `（補足）` を除去する（heat-1 相当）。
4. **タグ変換**: 各ラベルをフィルタタグ配列に変換する（heat-2 相当）。
5. **重複除去**: 同じタグが複数行から生成された場合は除去する。タグの順序は行の出現順を保つ。
6. `""` / `null` / `undefined` は `[]` を返す。

## あなたが決めること

JSDoc【契約】を書く4問への当てはめ:

- **正常時**: タグ配列の順序は何で決まるか。重複はあるか。
- **困った入力**: `null` / `undefined` / 空文字をどこで・どうやって弾くか。
- **しないこと**: 例外を投げるか?
- **暗黙の決め**: 重複除去に `Set` を使うか `includes` を使うか。それぞれ行の順序への影響は?
