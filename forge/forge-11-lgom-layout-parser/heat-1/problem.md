# extractRowLabel

難易度: ★☆☆

`lGomLayout` を改行分割した1行（例: `"3行目：TEL＆FAX"` や `"2行目：社名or店名（大）"`）から、フィルタタグの判定に使うラベル文字列を抽出する関数を書く。

> **背景**: CMS が出力する `lGomLayout` の値は `"1行目：郵便番号＆住所<br>2行目：社名or店名（大）<br>3行目：TEL＆FAX"` のような形式。`<br>` で分割すると各行は `"N行目：ラベル名"` または `"N行目：ラベル名（補足）"` になる。このうち実際にフィルタ判定に必要なのはラベル名の部分だけ。

## 実装する関数

`extractRowLabel` を自分で宣言・実装する。**引数・戻り値の型は自分で設計する**（`export` する名前は固定）。何を受け取り何を返すかは下の「変換仕様」と「合意済み仕様」から読み取って決める。

## 変換仕様

| 入力例 | 出力例 |
|--------|--------|
| `"3行目：TEL＆FAX"` | `"TEL＆FAX"` |
| `"1行目：郵便番号＆住所"` | `"郵便番号＆住所"` |
| `"2行目：社名or店名（大）"` | `"社名or店名"` |
| `"3行目：氏名（大）（ふりがな）"` | `"氏名"` |
| `"  住所  "` | `"住所"` |
| `""` | `""` |

## 合意済み仕様

- `\d+行目：` にマッチするプレフィックスを除去する（数字は1文字以上）。
- `（` から `）` までの全角カッコとその中身をすべて除去する。カッコが複数あってもすべて除去する。
- 先頭・末尾の空白を除去する（trim）。
- 上記の処理を **この順番で** 行う。

## あなたが決めること

JSDoc【契約】を書く4問への当てはめ:

- **正常時**: 除去後の文字列は何を表すか。呼び出し側はどう使うか。
- **困った入力**: 行番号プレフィックスがない行（`"住所"` 単体）や空文字はどう処理するか。
- **しないこと**: 例外を投げるか? 入力文字列を変更するか?
- **暗黙の決め**: 処理の順番を変えると結果が変わるケースはあるか?
