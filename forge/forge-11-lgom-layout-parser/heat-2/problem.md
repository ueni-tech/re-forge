# rowLabelToFilterTags

難易度: ★★☆

`extractRowLabel` で正規化したラベル文字列（例: `"TEL＆FAX"`、`"社名or店名"`）を、フィルタボタンの `data-filter-value` と一致するタグ文字列の配列に変換する関数を書く。

> **背景**: フィルタ UI のボタンは「TEL」「FAX」「住所」「会社名・店名」など10種類ある。CMS 側の文字列（`"社名or店名"`、`"役職＆氏名"` など）とボタン側の値（`"会社名・店名"`、`"氏名"` など）は表記が異なるため、変換テーブルが必要。`"TEL＆FAX"` だけは「TEL」と「FAX」の2ボタンに対応するため、2タグを返す特例がある。

## 実装する関数

`rowLabelToFilterTags` を自分で宣言・実装する。**引数・戻り値の型は自分で設計する**（`export` する名前は固定）。

## 変換仕様

| 入力ラベル | 出力タグ配列 |
|-----------|------------|
| `"TEL＆FAX"` | `["TEL", "FAX"]` |
| `"住所"` | `["住所"]` |
| `"郵便番号＆住所"` | `["住所"]` |
| `"社名or店名"` | `["会社名・店名"]` |
| `"部署or支店名"` | `["支社・支店・部署名"]` |
| `"氏名"` | `["氏名"]` |
| `"役職＆氏名"` | `["氏名"]` |
| `"TEL"` | `["TEL"]` |
| `"FAX"` | `["FAX"]` |
| `"メールアドレス"` | `["メールアドレス"]` |
| `"URL"` | `["ホームページ"]` |
| `"登録番号"` | `["インボイス番号"]` |
| `"キャッチコピー"` | `["キャッチコピー"]` |
| 未知のラベル | `[]` |

## 合意済み仕様

- `"TEL＆FAX"` は `["TEL", "FAX"]` の2要素を返す唯一の特例。
- `"郵便番号＆住所"` と `"住所"` はどちらも `"住所"` タグに変換する（郵便番号はフィルタ対象外）。
- `"役職＆氏名"` は `"氏名"` タグに変換する（役職はフィルタ対象外）。
- マッチしないラベルは `[]` を返す（例外は投げない）。
- パターンマッチは正規表現で行う。

## あなたが決めること

JSDoc【契約】を書く4問への当てはめ:

- **正常時**: 戻り値の配列は何を表すか。0件・1件・2件のそれぞれは何を意味するか。

- **しないこと**: 例外を投げるか?
